What is Secured In?

Secured In is an applicaion for iOS and Android designed to protect your data in an increasingly security-centric world. In the app you are able to save any text data, including passwords (the app even includes a password generator), as well as photos. Everything is encrypted with AES-256 when possible and with AES-128 as a fallback. AES-128 is the industry standard, and AES-256 is orders of magnitudes more difficult to break.

What technology is behind it?

The power of Secured In lies in the fact that the only way to view your data is to enter a password that you set. Using Password Based Key Derivation Function 2 (PBKDF2), a randomly generated salt, and a certain number of iterations, a key is derived based on the password that you enter. This key is what is used to encrypt the data. So, without the password, the key is not known and the data remains encrypted.

Secured In utilizes AES-256 to encrypt the data; AES-128 is an industry standard, and AES-256 is orders of magnitudes more difficult to break.

While an intutive system of encryption is used to secure user data, the problem gets slightly more complicated when syncing with a server and sharing with users is added. When a user (in this case let's say the user is Alice) opens the application, a challenge is sent to Alice's device to verify that it is in fact Alice. Only she would have her private key, so only she can actually complete the challenge. Sharing user data with others uses a combination of RSA and AES.
